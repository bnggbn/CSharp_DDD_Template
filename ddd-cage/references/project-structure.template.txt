    enums/
    services/             # pure domain services (*Service, PATH006); return results, no logging/persistence

  YourProject.Application/
    contracts/
      ports/              # only interfaces the application actually calls (logger, validators, repositories)
    use-cases/            # *UseCase.cs: nested Command/Query records only
    handlers/             # orchestrate: call domain service, then log/persist from the result
    validators/
    workflows/            # dispatch-only via IDispatcher
    behaviors/            # cross-cutting: validation, logging, exception handling
  YourProject.Infrastructure/
    configuration/

notes/
  - Business/domain services live in domain/services and return result value objects.
  - The application handler owns logging and the persist/skip decision, based on the result.
  - application/contracts/ports exposes only application-facing interfaces.
  - Interfaces consumed only by infrastructure live under infrastructure/<area>/abstractions.
